## 建站的心路历程

很久之前就想建一个自己的博客，奈何自己一直都没有时间。这两天闲下来了，正好想着能不能利用我手头的嵌入式开发板搭建一个自己的博客。于是就有了这个网站。我还有很多待解决的问题。这篇文章主要记录一下我的建站过程以及还有的问题。

<!-- more -->

### 建站第一步

我是跟着这篇文章 [在 Armbian 上搭建博客](https://blog.haibara.cn/archives/86) 开始的，因为我手头正好有一块 orangepi zero2 的板子，其核心是 ARM 架构的嵌入式开发板。之前我在 B 站搜索到的网站搭建教程好像大多都是基于 x64/x86 架构的芯片，导致我在建站过程中遇到了各种各样的问题。主要是我本身并不是做网站开发的，遇到问题很多时候也看不懂，所以每次做到一半就只能放弃。关于 PHP 建站的知识我可以说是完全没有。

跟着文章来搭建网站就顺利很多了。跟着网站我也学到了很多东西。首先是配置 LNMP，四个字母代表四种不同的技术。

L 代表的是 Linux 技术，因为我主要学嵌入式开发，所以对嵌入式 Linux 还是比较熟悉的，这一部分属于轻车熟路。

N 代表的是 nginx 技术，nginx 是一种 HTTP 服务器技术，主要用来提供网络服务。这一部分我完全不了解，毕竟之前完全没有接触过网络相关的知识，就算接触到过，由于完全没有用上，估计也忘得差不多了。

M 代表的是 MySQL 或者 MariaDB 技术，这是数据库技术。之前建站的时候也遇到过，当时就是卡在了数据库上，所以放弃了。好在这一次很顺利，在安装数据库支持时没有多少报错。

P 代表的是 PHP 技术，这是一种网站搭建技术。在我本科的时候听同学提到过这种技术，当时了解不多，而且那时候重心在单片机这样的嵌入式设备上，主要使用 C 语言，和寄存器打交道比较多，所以没有太关心。

在配置 LNMP 的过程中，除了下载相应的技术支持外，还需要做一些配置，主要是在 nginx 中打开对 PHP 的支持。这里面总共有五个地方需要修改，当时我就是少改了一处地方，导致最后 PHP 没有加载，排查错误排了一个多小时。之后是配置数据库，这一部分我也不是很理解，当时也出现了相关问题，最后我在 [垃圾堆](https://blog.csdn.net/weixin_42189863/article/details/125113978) 里找到了相应的解决方案。我在网站搭建方面属于是小白中的小白，所以在垃圾堆里刨食可能正好合适，哈哈哈。

### 第二步 管理数据库

管理数据库时，根据教程我使用的是 phpMyAdmin。它是一个用 PHP 开发的免费软件，可以让你在 Web 界面管理 MySQL/MariaDB 数据库，无需学习数据库命令，pma 是新人站长的福音。由于 pma 是基于 PHP 开发的，所以**安装 pma 跟搭建网站没什么区别**，同样需要你提前**搭建好 LNMP 环境**。

这一步很顺利，直接就可以显示管理界面。

### 最后一步

将下载好的 WordPress 包塞进 `/var/www/html` 文件夹中就完事了。搭建完网站后，再去搭建网页就相对容易了，尤其是使用 PHP 方法搭建的网站，目前使用的很多都是现成部分，所以操作起来还是比较容易。之后为 WordPress 创建数据库，再在 WordPress 中配置相应的数据库就行了。

目前我的密码是我常用密码中的一个，已经记录在浏览器中，所以问题不大。

最后加载 markdown 插件，完成！

## 待解决的问题

### 外网访问

我建立的 blog 网站，需要能被外网访问才有意义。如果不能外网访问，只能局域网访问，那就一点意义都没有。

### 本地浏览

虽然建立了数据库，我最好还是有办法在本地浏览我的文章，因为我不知道哪天服务器会崩，但是主题就在我身边。如果有办法通过本地访问文件的方法访问数据库，就能更好地保护本地数据。
